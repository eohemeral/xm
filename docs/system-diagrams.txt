3.2 数据表关系说明
--------------------------------------------------------------------------------

User (用户表)
│
├── 1:N ──► Post (帖子表)         一个用户可以发布多个帖子
├── 1:N ──► Comment (评论表)       一个用户可以发表多条评论
├── 1:N ──► StudyRecord (学习记录) 一个用户可以有多条学习记录
├── 1:N ──► Collection (收藏表)    一个用户可以收藏多个内容
└── 1:N ──► Like (点赞表)          一个用户可以点赞多个内容

Course (课程表)
│
├── 1:N ──► StudyRecord (学习记录) 一门课程可以被多个用户学习
└── 1:N ──► Collection (收藏表)    一门课程可以被多个用户收藏

Post (帖子表)
│
├── N:1 ──► User (用户表)          多个帖子属于一个用户
├── 1:N ──► Comment (评论表)       一个帖子可以有多条评论
└── 1:N ──► Like (点赞表)          一个帖子可以被多个用户点赞

Comment (评论表)
│
├── N:1 ──► Post (帖子表)          多条评论属于一个帖子
├── N:1 ──► User (用户表)          多条评论属于一个用户
└── 1:N ──► Like (点赞表)          一条评论可以被多个用户点赞

四、数据库表结构
================================================================================

4.1 user 表 - 用户表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       用户 ID               主键，自增
username       varchar(255)  用户名                唯一索引，非空
password       varchar(255)  密码 (MD5 加密)         非空
nickname       varchar(255)  昵称                  可空
email          varchar(255)  邮箱                  可空
phone          varchar(255)  手机号                可空
avatar         varchar(255)  头像 URL              可空
status         tinyint(1)    状态：1 启用/0 禁用      默认 1
createtime     datetime      创建时间              默认当前时间
updatetime     datetime      更新时间              可空

4.2 course 表 - 课程表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       课程 ID               主键，自增
title          varchar(255)  课程标题              非空
description    text          课程描述              可空
type           varchar(50)   课程类型              非空
level          varchar(50)   难度等级              非空
scenario       varchar(50)   应用场景              可空
cover          varchar(255)  课程封面 URL          可空
video_url      varchar(255)  视频地址              可空
content        text          课程内容              可空
view_count     int(11)       浏览量                默认 0
collect_count  int(11)       收藏量                默认 0
createtime     datetime      创建时间              默认当前时间
updatetime     datetime      更新时间              可空

4.3 post 表 - 帖子表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       帖子 ID               主键，自增
user_id        int(11)       发帖用户 ID           外键，索引
title          varchar(255)  帖子标题              非空
content        text          帖子内容              非空
type           varchar(50)   帖子类型              可空
view_count     int(11)       浏览量                默认 0
like_count     int(11)       点赞量                默认 0
comment_count  int(11)       评论量                默认 0
createtime     datetime      创建时间              默认当前时间
updatetime     datetime      更新时间              可空

4.4 comment 表 - 评论表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       评论 ID               主键，自增
post_id        int(11)       所属帖子 ID           外键，索引
user_id        int(11)       评论用户 ID           外键，索引
content        text          评论内容              非空
createtime     datetime      创建时间              默认当前时间
updatetime     datetime      更新时间              可空

4.5 study_record 表 - 学习记录表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       记录 ID               主键，自增
user_id        int(11)       用户 ID               外键
course_id      int(11)       课程 ID               外键
progress       int(11)       学习进度              默认 0
last_study_time datetime     最后学习时间          可空
createtime     datetime      创建时间              默认当前时间
updatetime     datetime      更新时间              可空

唯一索引：(user_id, course_id)

4.6 collection 表 - 收藏表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       收藏 ID               主键，自增
user_id        int(11)       用户 ID               外键
target_id      int(11)       目标 ID               可空
target_type    varchar(50)   目标类型：course/post  非空
createtime     datetime      创建时间              默认当前时间

唯一索引：(user_id, target_id, target_type)

4.7 like 表 - 点赞表
--------------------------------------------------------------------------------
字段名          类型          说明                  约束
────────────────────────────────────────────────────────────────────────
id             int(11)       点赞 ID               主键，自增
user_id        int(11)       用户 ID               外键
target_id      int(11)       目标 ID               可空
target_type    varchar(50)   目标类型：post/comment 非空
createtime     datetime      创建时间              默认当前时间

唯一索引：(user_id, target_id, target_type)

七、技术架构
================================================================================

客户端层
    ├─ Web 端 (Vue.js)
    ├─ 移动端 (扩展)
    └─ 管理端 (Vue.js)

表现层
    ├─ Vue Router (路由)
    ├─ Vuex (状态管理)
    ├─ Axios (HTTP 请求)
    └─ Element UI (UI 组件)

业务逻辑层
    ├─ Spring Boot (核心框架)
    ├─ Spring Security (安全框架)
    ├─ JWT (认证令牌)
    ├─ UserService (用户服务)
    ├─ CourseService (课程服务)
    └─ CommunityService (社区服务)

数据访问层
    ├─ MyBatis (ORM 框架)
    ├─ Mapper Interface (数据访问接口)
    └─ XML Mapping (SQL 映射文件)

数据存储层
    ├─ MySQL 5.7+ (关系数据库)
    └─ HikariCP (连接池)

九、总结
================================================================================

本系统是一个完整的在线办公学习平台，包含以下特点：

1. 功能模块完整
   - 用户管理：注册、登录、信息管理
   - 课程学习：浏览、学习、记录进度
   - 社区交流：发帖、评论、点赞
   - 个人中心：我的课程、帖子、收藏
   - 后台管理：用户、课程、内容管理

2. 数据库设计合理
   - 7 个核心数据表
   - 规范的外键关系
   - 多态关联设计
   - 完善的索引优化

3. 技术架构先进
   - 前后端分离
   - RESTful API
   - JWT 认证
   - 响应式设计

4. 性能优化考虑
   - 连接池管理
   - 索引优化
   - 计数缓存
   - 负载均衡
