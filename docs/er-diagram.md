# 系统 E-R 图

## 一、实体列表

系统共有 **7 个实体**：

1. **用户（User）**
2. **课程（Course）**
3. **学习进度（CourseProgress）**
4. **评论（Comment）**
5. **社区帖子（Community）**
6. **收藏（Favorite）**
7. **点赞（Like）**

---

## 二、实体详细说明

### 1. 用户实体（User）

**实体说明**：系统使用者，包括管理员和普通用户

**属性**：
- 用户 ID（主键）
- 用户名
- 密码（MD5 加密）
- 姓名
- 邮箱
- 电话
- 头像 URL
- 角色（管理员/普通用户）
- 创建时间
- 更新时间

**关联关系**：
- 1:N → 学习进度
- 1:N → 评论
- 1:N → 社区帖子
- 1:N → 收藏
- 1:N → 点赞

---

### 2. 课程实体（Course）

**实体说明**：在线学习内容

**属性**：
- 课程 ID（主键）
- 课程名称
- 课程描述
- 封面图片 URL
- 课程内容
- 创建时间

**关联关系**：
- 1:N → 学习进度
- 1:N → 评论
- 1:N → 收藏（通过多态关联）
- 1:N → 点赞（通过多态关联）

---

### 3. 学习进度实体（CourseProgress）

**实体说明**：记录用户学习课程的进度

**属性**：
- 记录 ID（主键）
- 用户 ID（外键 → 用户表）
- 课程 ID（外键 → 课程表）
- 学习进度百分比
- 最后学习时间

**关联关系**：
- N:1 → 用户
- N:1 → 课程

---

### 4. 评论实体（Comment）

**实体说明**：用户对课程的评价

**属性**：
- 评论 ID（主键）
- 用户 ID（外键 → 用户表）
- 课程 ID（外键 → 课程表）
- 评论内容
- 创建时间

**关联关系**：
- N:1 → 用户
- N:1 → 课程

---

### 5. 社区帖子实体（Community）

**实体说明**：用户发布的讨论帖

**属性**：
- 帖子 ID（主键）
- 用户 ID（外键 → 用户表）
- 帖子标题
- 帖子内容
- 创建时间

**关联关系**：
- N:1 → 用户
- 1:N → 收藏（通过多态关联）
- 1:N → 点赞（通过多态关联）

---

### 6. 收藏实体（Favorite）

**实体说明**：用户收藏的课程或帖子（多态设计）

**属性**：
- 收藏 ID（主键）
- 用户 ID（外键 → 用户表）
- 收藏类型（course/community）
- 项目 ID（课程 ID 或 帖子 ID）
- 创建时间

**关联关系**：
- N:1 → 用户
- 多态关联 → 课程 或 社区帖子

---

### 7. 点赞实体（Like）

**实体说明**：用户点赞的课程或帖子（多态设计）

**属性**：
- 点赞 ID（主键）
- 用户 ID（外键 → 用户表）
- 点赞类型（course/community）
- 项目 ID（课程 ID 或 帖子 ID）
- 创建时间

**关联关系**：
- N:1 → 用户
- 多态关联 → 课程 或 社区帖子

---

## 六、设计特点

### 1. 多态关联设计

**收藏表**和**点赞表**采用多态设计：
- 通过 `类型` 字段区分关联对象（课程/帖子）
- 通过 `项目 ID` 字段存储对应对象的 ID
- **优点**：用一张表实现对多个实体的关联
- **缺点**：不能直接使用外键约束

### 2. 核心实体

- **用户**和**课程**是系统的两个核心实体
- 其他 5 个表都是关联表，记录用户行为

### 3. 完整的行为追踪

所有用户行为（学习、评论、收藏、点赞）都有记录，形成完整的用户行为追踪链。

---

## 七、总结

- **实体数量**：7 个
- **核心表**：2 个（用户表、课程表）
- **关联表**：5 个（学习进度、评论、社区、收藏、点赞）
- **特殊设计**：收藏表和点赞表采用多态关联
- **关系类型**：主要是一对多关系，用户和课程之间通过学习进度表实现多对多关系
